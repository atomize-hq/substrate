## Assumptions

These are the assumptions this remediation spec relies on. If any are wrong, stop before implementation and revise the slice rather than guessing in code.

1. The frozen architectural floor from 28.5, 29.75, and 30 is authoritative: blank-prompt control semantics are no longer an allowed forward path for Substrate attach behavior.
2. Packet 4's persisted-truth planning work in `auto_attach.rs` is directionally correct and should be preserved where it does not require forbidden prompt-free fresh attach execution.
3. The shared `hidden_owner_helper` launch seam may remain as internal plumbing if it is not carrying prompt-free fresh attach semantics back into the public architecture.
4. There is currently no sanctioned non-prompt fresh attach primitive for either Codex or Claude that has been reviewed as compatible with the frozen Substrate contract.
5. This slice is remediation-only. It must remove the regression and restore truthful docs/tests without reopening the larger slice-31 product scope.

## Objective

Freeze the smallest honest remediation after Packet 4:

1. preserve continuity-first attach and persisted host-attach truth planning;
2. remove prompt-free fresh control execution from the runtime architecture;
3. make fresh-needed attach paths fail closed until a separately sanctioned non-prompt mechanism exists;
4. realign docs, tests, and closeout language with the frozen 28.5/29.75/30 direction.

## Code Style

Use explicit fail-closed diagnostics and keep forbidden runtime paths impossible rather than merely "known-bad."

Example contract style:

```rust
match continuity_session_id {
    Some(session_id) => prompt_fulfillment.run_attach_control(session_id).await,
    None => Err(RuntimeBootstrapFailure {
        exit_code: 1,
        message: "unsupported_attach_mode: fresh control-only attach is not sanctioned in this slice".to_string(),
    }),
}
```

Conventions:

1. Prefer explicit rejection over fallback when the remaining fallback would violate a frozen architectural rule.
2. Keep persisted `HostAttachContract` truth authoritative even when attach execution must fail closed.
3. Preserve the distinction between "planning says fresh would be required" and "runtime is allowed to execute fresh attach."
4. Do not let backend-specific wrapper affordances silently redefine the Substrate contract.

## Testing Strategy

This remediation must prove both removal and preservation:

1. Unit tests prove control-only attach startup no longer permits fresh attach fallback when continuity is absent.
2. Unit and integration tests preserve continuity-backed attach behavior where it is still sanctioned.
3. Automatic attach tests distinguish:
   - continuity available and allowed,
   - continuity required but missing,
   - fresh would be required but is not sanctioned in this slice.
4. Regression tests prove no prompt-free fresh attach helper exists for Codex or Claude in the active runtime path.
5. Slice-31.25 docs and corrective closeout explicitly supersede Packet 4's planner/bootstrap-only prompt-free fresh attach overclaim for shipped behavior.

## Boundaries

- Always:
  - Preserve persisted attach-truth planning from `HostAttachContract`.
  - Preserve continuity-first attach where a valid continuity selector exists.
  - Fail closed before backend launch when fresh attach would be required but no sanctioned mechanism exists.
  - Keep this corrective slice self-contained instead of rewriting slice-31 planning artifacts.
- Ask first:
  - Sanctioning any new non-prompt fresh attach primitive.
  - Reopening Packet 4 as "partially shipped fresh attach" instead of explicitly remediating it.
  - Renaming the slice if reviewers want it folded into a different numbering scheme.
- Never:
  - Reintroduce prompt-free control execution by empty prompt, `.no_prompt()`, or synthetic bootstrap language.
  - Treat backend wrapper acceptance as architectural authorization.
  - Leave Packet 4 docs claiming compliance once the runtime path is known to violate 28.5/30.

## Success Criteria

This remediation slice is complete only when all of the following are true:

1. No active control-only attach runtime path launches a prompt-free fresh attach for Codex or Claude.
2. Continuity-backed attach remains available where persisted continuity truth is valid.
3. Fresh-needed attach cases fail closed before backend launch with explicit explanation-ready diagnostics.
4. Automatic attach still plans from persisted host-attach truth, but does not execute a forbidden fresh attach runtime path.
5. Tests no longer encode "fresh attach without continuity is valid startup behavior" as an allowed outcome for this slice.
6. Slice-31.25 spec, plan, tasks, and corrective closeout explicitly supersede Packet 4's planner/bootstrap-only prompt-free fresh attach overclaim for shipped behavior.
7. Any future fresh attach work is explicitly deferred behind a separate reviewed slice that introduces a sanctioned non-prompt attach mechanism.
8. This corrective work stands on its own as slice `31.25`; it does not require rewriting slice-31 planning docs to be valid.
